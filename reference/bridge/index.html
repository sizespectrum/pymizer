    R Bridge API

    The bridge API covers the lower-level runtime layer that connects Python to the embedded R session.

    Most users only need:

    • get_environment() to inspect the shared bridge instance
    • runtime_diagnostics() to troubleshoot setup problems
    • evaluate_versions() and CompatibilityReport to reason about supported runtime combinations

    You can also use MizerREnvironment directly when you need to inspect raw R objects or call additional exported mizer functions.

    pymizer._bridge

    MizerError

    Bases: RuntimeError

    Raised when the R bridge cannot complete an operation.

    This usually indicates one of three problems:

    • rpy2 could not be imported or initialised against the local R install
    • the R package mizer is unavailable
    • an R-side function call failed

    CompatibilityReport(versions, minimum_versions, issues) dataclass

    Compatibility report for the active Python and R runtime stack.

    Attributes:

    Name Type Description
    versions dict[str, str]

    Detected versions for the current Python, R, rpy2, and mizer runtime stack.
    minimum_versions dict[str, str]

    Minimum versions expected by pymizer.
    issues list[str]

    Human-readable compatibility problems, if any.

    ok property

    Return whether the runtime environment passes compatibility checks.

    MizerREnvironment(package_name='mizer') dataclass

    Thin wrapper around the embedded R session used by pymizer.

    Most users should access the shared singleton via :func:get_environment rather than instantiating this class directly.

    Examples:

    import pymizer as mz

env = mz.get_environment()
print(env.versions())

    class:CompatibilityReport.

    get_environment()

    Return the singleton R environment used by pymizer.

    Returns:

    Type Description
    MizerREnvironment

    The shared :class:MizerREnvironment instance.

    Examples:

print(env.compatibility_report().ok)

    runtime_diagnostics()

    Return environment diagnostics without requiring access to internals.

    The returned dictionary is designed for quick troubleshooting in notebooks, scripts, or issue reports.

    Returns:

    Type Description
    dict[str, Any]

    A dictionary describing:
    dict[str, Any]
    • whether rpy2 imported successfully
    dict[str, Any]
    • minimum supported versions
    dict[str, Any]
    • detected runtime versions when available
    dict[str, Any]
    • overall compatibility status
    dict[str, Any]
    • any issues that were found

    Examples:

    import pymizer as mz

diagnostics = mz.runtime_diagnostics()
print(diagnostics["compatibility"])
